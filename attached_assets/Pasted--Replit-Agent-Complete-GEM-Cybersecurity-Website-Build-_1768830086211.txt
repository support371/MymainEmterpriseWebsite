## ğŸ”§ BACKEND API ROUTES

### API STRUCTURE

**Route 1: Contact Form Submission**

```
POST /api/contact

Request Body:
{
  "name": "string",
  "email": "string",
  "company": "string",
  "phone": "string (optional)",
  "service": "string",
  "message": "string"
}

Response:
{
  "success": boolean,
  "message": "string"
}

Logic:
1. Validate input (email format, required fields)
2. Send email notification to admin
3. Send confirmation email to user
4. Store in database (optional)
5. Return success response

Error Handling:
- 400: Invalid input
- 500: Server error
```

**Route 2: Newsletter Signup**

```
POST /api/newsletter

Request Body:
{
  "email": "string"
}

Response:
{
  "success": boolean,
  "message": "string"
}

Logic:
1. Validate email format
2. Check if already subscribed
3. Add to email list (Mailchimp/SendGrid)
4. Send welcome email
5. Return success

Integration: Mailchimp or SendGrid API
```

**Route 3: Assessment Booking**

```
POST /api/assessment

Request Body:
{
  "email": "string",
  "company": "string",
  "service": "string",
  "preferredDate": "string (optional)"
}

Response:
{
  "success": boolean,
  "calendlyUrl": "string (if using Calendly)",
  "message": "string"
}

Logic:
1. Validate input
2. Create Calendly event or
3. Store booking request
4. Send confirmation email
5. Notify sales team
6. Return booking confirmation
```

**Route 4: Resource Download**

```
POST /api/download

Request Body:
{
  "email": "string",
  "resourceId": "string"
}

Response:
{
  "success": boolean,
  "downloadUrl": "string"
}

Logic:
1. Validate email
2. Add to newsletter list
3. Generate secure download link
4. Send download email
5. Track download analytics
```
